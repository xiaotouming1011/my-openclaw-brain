# FaceWake - 本地摄像头表情触发器设计思路文档

## 💡 一、产品定位与核心价值
**“用一个眼神唤醒你的 AI 助理”**

FaceWake 是一款运行在本地的静默交互工具，通过计算机视觉技术（Computer Vision）捕捉用户的面部表情变化（如挑眉、眨眼、微笑），将其转化为系统级指令（如唤醒 OpenClaw 对话、静音、截屏等）。

*   **绝对隐私安全**：全部在本地 CPU 运算，视频帧“阅后即焚”，零数据上云。
*   **零噪音交互**：完美弥补“语音唤醒（VoiceWake）”在办公室、深夜等场景下的局限性。
*   **蓝海创新**：跳出传统的“看图说话”，实现真正的“基于持续动作流的指令触发”。

---

## 🛠️ 二、核心技术架构设计（Python栈）

### 1. 👁️ 视频流捕获 (Eyes)
*   **工具选型**：`OpenCV-Python` (`cv2`)
*   **实现方式**：调用 macOS 摄像头底层权限，以较低的帧率（如 5-10 FPS，节省性能）持续拉取画面。

### 2. 🧠 表情识别与特征提取 (Brain)
*   **核心引擎**：Google `MediaPipe (Face Mesh)` 和 `Tasks API`
*   **技术原理**：一种极度轻量化的 3D 面部网格/手部骨骼映射模型，能够在普通电脑 CPU 上实时输出 3D 关键点坐标。相比于调用百兆级别的大模型，它的速度快了几个数量级。

### 3. ⚡ 信号判定逻辑 (Nervous System)
*   **几何判定法与状态机**：通过计算特定关键点之间的相对距离变化来识别手势或表情。
    *   *判断【挑眉】 = 计算“眉峰”与“瞳孔中心”的垂直距离。*
    *   *判断【数字手势】 = 识别伸展手指的数量。*

### 4. ✋ 动作执行与下发 (Hands)
*   **接入方式**：当判定触发后（例如某个手势保持 2 秒），Python 脚本通过 `subprocess` 调用 `openclaw system event`，附带特定的语音唤醒标识头，向本地 Gateway 发送系统级请求。

## ⚠️ 三、关键难点与解决方案

### 难点 1：如何防止自然表情“误触”（Debounce）
*   **解决方案（时间窗 Time Threshold）**：引入持续时间判定。例如：“单手比划 2”必须持续 **2.0 秒以上**，才算作一次有效指令，晃动被滤除。

### 难点 2：高强度监听导致的系统发热与高负载
*   **解决方案**：大幅降低抽帧率。人类的表情/手势变化较慢，将视频流降至 **5 FPS**，完全足够识别，并将 CPU 占用降到最低。

### 难点 3：光线不足的干扰
*   **解决方案**：加入“环境光亮度计算”，当平均亮度低于阈值时，停止 MediaPipe 计算以节省算力。
