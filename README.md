# my-openclaw-brain

这是一个 **Agent 记忆仓库**，用于持久化保存与 `Anji (Digital Form)` 相关的历史上下文。  
当前仓库设计为 **只保留记忆数据**，不承载运行代码、技能实现或临时工具脚本。

## 仓库作用

- 作为跨会话、跨机器的长期记忆来源。
- 让未来启动的 agent 能快速理解“我是谁、做过什么、接下来该如何延续”。
- 通过 Git 历史保留关键决策与事件演进轨迹。

## 当前结构

- `memory/`：唯一核心目录。
  - `YYYY-MM-DD*.md`：会话日志、事件记录、阶段总结。
  - 其他 `*.md`：专题计划或迁移说明（如 dashboard 迭代计划）。
  - `heartbeat-state.json`：周期性检查状态（如最后检查时间）。

## 给未来 Agent 的读取顺序（建议）

每次进入仓库后，按以下顺序读取：

1. 最近两条时间相关的 `memory/*.md`
2. 与当前任务最相关的专题文档（例如 `memory/dashboard-iteration-plan.md`）
3. `memory/heartbeat-state.json`（仅在执行 heartbeat/巡检任务时）

> 目标：最小上下文读取，优先相关性，控制 token 成本。

## 维护规则

- 新增记忆优先写入 `memory/`，不要散落到仓库根目录。
- 避免写入敏感信息（密钥、口令、私人凭据）。
- 记录“决策 + 原因 + 结果”，不要只写结论。
- 文件名尽量时间化、可检索（如 `2026-03-01-xxx.md`）。

## 非目标（明确边界）

此仓库 **不用于**：

- 存放可执行程序或服务代码
- 保存依赖锁文件、前端构建产物
- 充当通用项目 monorepo

如需代码与配置，请放在独立仓库；这里仅保留“记忆与上下文”。

## 协作约定

- 修改记忆后应及时提交，保证远程状态可恢复。
- 提交信息建议使用动词开头，例如：
  - `docs(memory): add 2026-03-01 session summary`
  - `chore(memory): archive obsolete heartbeat notes`

---

如果你是新启动的 agent：
先读取 `memory/`，再回答用户问题；不要在未读取近期记忆前假设上下文。
