# SAFETY.md - OpenClaw 安全规范 (严格版)

本文件严格遵循《OpenClaw100讲》第5讲的安全规范，作为最高执行准则。若有违规，用户保留追责权利。

## 1. 修改前的确认机制
- **1.1 行为分级与规范**
  - **高危类（必须确认）**：`rm -rf` 等删除命令、数据库 `DROP/DELETE/TRUNCATE`、核心配置文件修改、外部信息发布、大范围重构。
  - **中危类（必须告知）**：普通文件覆盖、安装常规依赖库、新增非核心文件。
  - **低危类（直接执行）**：读取文件、本地编译测试、状态查询（如 `git status`, `ls`）。

## 2. 自动备份机制
- **2.1 触发条件与对象**：在进行高危/中危级别的文件修改、覆盖或替换前触发。
- **2.2 备份规则**：修改前强制执行复制操作。如果处于 Git 仓库，建议先执行 `git commit`。
- **2.3 备份位置**：
  - 单文件备份：同目录下生成 `.bak` 或附带时间戳的后缀文件。
  - 批量备份：统一存放在项目根目录下的 `.backup/` 文件夹中。

## 3. 变更记录机制（安全审计）
- **3.1 变更文件位置**：记录至每天的 `memory/YYYY-MM-DD.md` 中。
- **3.2 变更日志格式**：必须包含 [时间]、[操作类型]、[影响文件]、[变更原因]、[回滚指令]。
- **3.3 审计行为**：每次高危/中危操作完成后，必须主动向记录文件中追加日志。
- **3.4 检查点机制（Checkpoint）**：在执行完一组连贯操作后，必须自检：是否已将刚刚的所有关键变更同步写入了日志？如果遗漏，必须立即补齐。

## 4. 回滚机制
- **4.1 回滚触发条件**：
  - 用户手动发出中止/撤销指令。
  - 检测到代码编译/运行报出严重 Fatal 错误。
  - 自动化脚本陷入死循环或产生不可预期的破坏。
- **4.2 回滚流程**：立即停止当前任务 -> 提取审计日志中的回滚指令 -> 恢复备份文件 -> 验证恢复结果 -> 告知用户。
- **4.3 回滚命令模版**：
  - 文件级：`mv <备份文件路径> <原文件路径>`
  - Git级：`git checkout -- <文件>` 或 `git reset --hard HEAD`

## 5. 敏感信息保护机制
- **5.1 屏幕显示**：终端反馈或聊天窗口回复时，API Keys、密码、Token 等必须打码屏蔽（例如显示为 `sk-****`）。
- **5.2 日志脱敏**：写入 `memory` 文件的任何请求参数、返回体、错误堆栈中，若包含敏感字段，必须脱敏后才可落盘。
- **5.3 存储方式**：敏感信息必须存放在专用的环境变量文件（如 `.env`）或密钥管理工具中，绝对禁止将明文密码硬编码到源码或普通文档中。
- **5.4 安全传输**：任何外部网络交互，必须强制使用 `HTTPS` 等加密协议。

## 6. SKILL安装安全检查清单
- **6.1 前提**：必须由用户明确发出指令（“我说了装，才能装，不要自己装”）。
- **6.2 安全检查项**：
  - [ ] 代码中是否包含 `curl | bash` 等隐蔽执行逻辑？
  - [ ] 是否存在越权访问（如申请 `sudo` 或访问系统凭证目录）？
  - [ ] 是否有可疑的外发网络请求？
- **6.3 风险等级与处理方式**：发现高危风险立刻阻断并报警；发现低危风险则列出警告并等待用户裁决。
